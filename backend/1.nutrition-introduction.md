# Cahier des Charges — API de Gestion de Régime Alimentaire

## Introduction

API de gestion de régime alimentaire destinée à un seul type d'utilisateur : l'utilisateur final qui souhaite gérer son alimentation selon son objectif personnel.

L'utilisateur a un profil physiologique (âge, poids, taille, sexe) et un niveau d'activité physique. Ces données déterminent ses besoins caloriques. Sur cette base, il crée et suit une **diète** — concept central de l'application.

L'utilisateur peut avoir trois objectifs :
- **Perte de poids** : apport calorique inférieur aux besoins
- **Maintien** : apport calorique égal aux besoins
- **Prise de masse** : apport calorique supérieur aux besoins

L'activité physique influence les besoins caloriques mais ne modifie pas la structure de la diète.
